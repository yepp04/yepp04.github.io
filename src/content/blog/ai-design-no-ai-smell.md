## "이거 AI로 만들었죠?"

AI로 웹사이트를 만들면 기능은 어느 정도 돌아갑니다. 그런데 디자인이 문제입니다. 뭔가 엉성하고, 어디서 본 듯하고, 뻔합니다. 괜찮은 서비스를 만들어 놓고도 디자인 때문에 출시를 망설이는 사람이 꽤 많습니다.

왜 이런 일이 생길까요?

AI는 학습 데이터의 **확률적 평균**을 선호합니다. 가장 널리 퍼진 디자인, 가장 안전한 폰트, 가장 무난한 색상을 골라냅니다. 파란색-보라색 그라데이션, 기본 고딕체, 뻔한 카드 레이아웃. AI는 실패하지 않기 위해 "지루함의 영역"으로 도망칩니다.

이것이 바로 "평균의 저주"입니다.

이 글에서는 평균의 저주를 깨는 구체적인 방법들을 정리합니다.

## 1. 디자인 시스템부터 세워라

AI 디자인이 엉성한 근본 원인은 **디자인 시스템의 부재**입니다. 색상, 폰트, 버튼 스타일, 간격 — 이런 기준 없이 AI에게 맡기면 매번 다른 기준으로 요소를 생성합니다. 통일감이 깨지고 엉성한 느낌이 나는 건 당연합니다.

### 방법 1: 레퍼런스 캡처 (초보 추천)

가장 간단한 방법입니다. 마음에 드는 웹사이트를 **스크린샷으로 캡처**해서 AI에게 전달합니다.

실제로 토스(Toss) 홈페이지를 캡처해서 Claude Code에 드래그 앤 드롭하고, "이 이미지의 디자인 기준을 분석하고 서비스 소개 홈페이지를 만들어 달라"고 요청하면 — AI가 자동으로 레이아웃 패턴, 타이포그래피, 색상 시스템, 컴포넌트 스타일을 분석한 디자인 문서를 작성합니다. 버튼 호버 인터랙션, 모바일 반응형까지 자동 처리됩니다.

현업에서는 말로 설명하는 부분이 거의 없습니다. **이미지 전달 + 실수 예상 부분만 간단히 설명**하면 됩니다. MVP 수준이라면 이 방법만으로 충분합니다.

### 방법 2: 검증된 디자인 키트 활용

세계적인 디자인 팀이 만들어 둔 검증된 기준을 가져다 쓰는 방법입니다. **shadcn/ui**, **MUI** 같은 디자인 키트는 무료로 사용할 수 있고, 컴포넌트의 완성도와 통일성이 높습니다.

```
"전체적인 디자인을 shadcn/ui로 변경해 줘"
```

이 한 줄이면 버튼, FAQ, 아이콘 등 컴포넌트의 컬러, 둥글기, 간격이 한 번에 통일됩니다. 분위기 변경도 가능합니다. "레트로한 디자인을 적용하고 싶은데 사람들이 많이 쓰는 디자인 형태를 조사하고 적용해 줘"라고 하면 NES.css(8비트 레트로) 같은 것을 제안하기도 합니다.

### 방법 3: 디자인 원칙 JSON

이 방법이 가장 체계적입니다. Dribbble이나 Behance에서 도메인에 맞는 레퍼런스를 찾고, AI에게 **디자인 원칙을 JSON으로 추출**시킵니다.

핵심 프롬프트:

```
"해당 페이지의 UX/UI 디자인 원칙을 JSON 형태로 정리해 줘.
콘텐츠와 디자인 요소를 연결지어서 맥락을 포함한 원칙으로 정리해 줘"
```

여기서 "콘텐츠와 디자인 요소를 연결"이 핵심입니다. 단순히 "파란색을 써라"가 아니라 "연간 수익 비교 → 막대 그래프가 적합"처럼, **무엇을 보여주는가**와 **어떻게 보여주는가**를 연결짓는 겁니다.

JSON으로 정리하는 이유는 토큰 절약입니다. 조사나 불필요한 텍스트가 빠지니까요. 이 JSON을 프로젝트 디렉토리에 넣고 Claude에게 참조시키면, "AI가 한 것 같지 않은, 사람이 한 것 같은 퀄리티"가 나옵니다.

## 2. 평균에서 탈출하라

디자인 시스템을 세우는 것만으로는 부족합니다. AI가 기본적으로 선택하는 "뻔한 요소들"을 의도적으로 피해야 합니다.

### 폰트: 국룰 금지

99%의 AI 웹사이트가 맑은 고딕, 본고딕 같은 기본 폰트를 씁니다. 이것만 바꿔도 **무료 템플릿 느낌을 80% 없앨 수 있습니다.**

- **프리텐다드(Pretendard)**: 깔끔하고 모던한 한글 폰트
- **리디바탕**: 감성적인 본문용 폰트

### 컬러: 불편한 색상 쓰기

AI는 호불호 없는 파란색/보라색 그라데이션을 사랑합니다. 과감해지세요.

- 형광 라임 (당근마켓처럼)
- 딥 오렌지 (무신사처럼)
- 브랜드 개성이 뚜렷한 컬러를 선택하면 "AI가 만든 것 같다"는 느낌이 확 줄어듭니다

### 아이콘: SF Symbols 탈출

Apple 기본 아이콘(SF Symbols)은 모든 앱이 사용합니다. 대안으로 **Hero Icons**(특히 thin 버전)을 추천합니다. 미니멀하고 클린하면서 무료입니다.

아이콘 선택보다 더 중요한 건 **일관성**입니다. 한 스타일을 고르면 전체에 일관되게 적용하세요. 예외가 있다면 활성/비활성 상태 — thin(비활성), filled(활성)으로 구분하는 정도입니다.

## 3. 디자인 DNA를 이식하라

일회성이 아니라, **모든 프로젝트에 같은 느낌**을 적용할 수 있다면? 이것이 디자인 DNA 주입입니다.

### Step 1: Google Stitch로 나만의 디자인 표준 만들기

Google Stitch는 구글의 무료 웹사이트 디자인 툴입니다. 여기서 웹사이트를 만드는 게 목적이 아닙니다. **내가 원하는 디자인 시스템을 레퍼런스로 만드는 것**이 목적입니다.

프롬프트와 드래그 앤 드롭으로 3가지 시안 중 마음에 드는 것을 선택하고, 폰트, 색상, 간격, 레이아웃을 반복해서 다듬습니다.

### Step 2: Claude 스킬로 학습시키기

Stitch에서 만든 HTML + 스크린샷을 다운로드해서 Claude의 **스킬(Skill)** 기능으로 학습시킵니다. 프롬프트에 물리법칙 적용 원칙도 포함하면 좋습니다:

- 요소 등장 시 0.05초 시차
- 마우스 올렸을 때 살아있는 듯한 반응
- 부드러운 스크롤 전환

### Step 3: 모든 프로젝트에 적용

스킬 없이 "SNS 프로필 페이지 만들어 줘" → 흔하디 흔한 평범한 디자인.
스킬 적용 후 같은 요청 → 개성 있고 일관된 브랜드 느낌, 세세한 애니메이션 디테일.

한 번 만들어 둔 디자인 DNA는 포트폴리오, 제품 소개, SNS 배너 등 모든 곳에 동일하게 적용됩니다. 애플처럼 모든 제품이 같은 디자인 언어를 쓰는 것 — 이것이 개인 브랜딩의 시작입니다.

## 4. Front-end Design Skill 활용

X(Twitter)에서 235,000+ 뷰를 기록한 바이럴 포스트에서 소개된 **front-end design skill**도 강력한 도구입니다.

Claude Code에 이 스킬을 설치하면:

- **블랙, 화이트, 원 컬러** 사용 (좋은 디자인의 기본)
- 스크롤 시 글로우 효과, 페이드인 애니메이션, 호버 이펙트
- 섹션 배경에 서브틀한 그리드 패턴
- 일관된 패딩과 스타일링

Dribbble 스크린샷과 함께 사용하면 더 강력합니다. "Use the front-end design skill to improve the design following the attached screenshot"이라고 하면 — 하드 그림자가 소프트 블러로 바뀌고, 각진 엣지가 라운드 처리되고, 모던 산세리프 폰트에 소프트 파스텔 그라데이션이 적용됩니다.

> "Front-end design skill 하나로 보라색 그라데이션 지옥에서 탈출할 수 있다."

## 5. 섹션별로 분해하고, 반복 개선하라

### 섹션별 접근

"사이트 만들어 줘"보다 **섹션 단위로 요청**하는 것이 품질이 훨씬 좋습니다.

- 히어로 섹션만 마음에 드는 사이트에서 캡처해서 교체
- 레퍼런스 소스: 해외 **Mobbin**, 국내 **GD Web**

단, 무작정 캡처해서 넣지 마세요:

1. 캡처한 이미지를 AI에게 "현재 사이트와 잘 어울리는지" 분석 요청
2. 기존에 만들어 둔 컴포넌트(버튼, 폰트 등)만을 사용해서 작업 요청
3. **기존 디자인 시스템을 망가뜨리지 않는 것이 핵심**

복잡한 애니메이션도 마찬가지입니다. 원샷으로 요청하면 실패합니다. **서브 동작 단위로 분해**하세요:

- "화살표를 체크마크로 회전"
- "배경을 마이크에서 확장"
- "텍스트 페이드인/아웃"

각각 요청하면 AI가 실수 없이 정교한 결과물을 만들어냅니다.

### 반복 개선

첫 결과물에 만족하지 마세요. **5~6번 반복 개선**하면 품질이 극적으로 올라갑니다. "디자인을 한 번 더 개선해 줘"만으로도 충분합니다. 컨텍스트 윈도우에 대화가 쌓일수록 AI가 맥락을 더 잘 이해하기 때문입니다.

## 6. 도구를 조합하라

단일 도구보다 여러 도구를 조합하면 훨씬 효과적입니다.

### 디자인 특화 도구 조합 (Gemini 3.0 Pro 기준)

| 도구 | 역할 |
|------|------|
| **Unicorn Studio** | 모션 배경/애니메이션 생성 (오로라, 메탈, 반응형 라이트) |
| **Super Hero** | 히어로 섹션 디자인 라이브러리 |
| **Iconify** | 무료 오픈소스 아이콘 (Solar 아이콘 추천) |
| **Bento Grids** | 중단부/하단부 그리드 디자인 라이브러리 |
| **AI Studio** | 위 요소들을 조합한 코드 생성 |

이 5가지를 조합하면 프로 디자이너 에이전시가 며칠 걸릴 결과물을 뽑아낼 수 있습니다.

### 실전 워크플로우

1. Unicorn Studio에서 모션 배경 선택 → Embed 코드 복사
2. Super Hero에서 히어로 섹션 레퍼런스 캡처
3. Iconify에서 Solar 아이콘 URL 확보
4. Bento Grids에서 하단부 그리드 디자인 캡처
5. AI Studio에 모두 전달 → 통합 결과물 생성
6. **5~6번 반복 개선**

### AI 디자인 도구 비교

현재 사용할 수 있는 주요 AI 디자인 도구를 정리하면:

| 도구 | 가격 | 핵심 강점 | 백엔드 | 적합한 용도 |
|------|------|-----------|--------|-------------|
| Google Stitch | 무료 | 빠른 시안 생성 | X | 디자인 시안/레퍼런스 |
| Figma Sites | $20/월 | 디자인→배포 일체형 | X | 랜딩/포트폴리오 |
| Figma Make | $20/월 | 디자인→코드 정확도 | Supabase | 프로토타입/MVP |
| Ready | $20/월 | 데이터 수집 내장 | 내장 DB | 폼/설문/예약 |
| Lovable | $25/월 | Supabase 통합 | Supabase | 풀스택 MVP |
| Framer | 별도 | CMS, SEO, 애니메이션 | X | 마케팅 웹사이트 |

시안은 Stitch, 프로토타입은 Figma Make, 풀스택 MVP는 Lovable — 목적에 맞는 도구를 선택하세요.

### Pencil + Claude Code

IDE 안에서 피그마 같은 무한 캔버스를 제공하는 **Pencil**도 주목할 만합니다. Claude Code와 MCP로 연동되어, **디자인을 먼저 확정한 후 코딩**으로 넘어가는 워크플로우가 가능합니다.

핵심 프롬프트: "코딩하기 전에 펜슬 MCP를 사용해서 디자인부터 먼저 진행" — 이 문구가 없으면 Claude Code가 바로 코딩을 시작합니다.

## 프로 디자이너의 관점

여기까지 읽으면 "AI가 다 해주네"라고 생각할 수 있지만, 프로 디자이너 Rachel How의 관점은 다릅니다.

> "나는 AI가 나 대신 디자인하는 것을 거부한다."

그녀는 AI로 디자인을 **생성**하는 것은 거부합니다. 대신 AI를 디자인 **프로세스를 가속하는 도구**로 활용합니다:

- **Granola**: 회의 자동 메모, 액션 포인트 정리
- **ChatGPT**: 클라이언트 브리프 해석, 도메인 지식 학습, 디자인 비평
- **Super Whisper**: 음성→텍스트 변환 (타이핑보다 빠름)
- **Claude Code**: 데이터 시각화, 풀 컨트롤이 필요한 프로토타이핑

AI가 모든 것을 대체하는 시대에, 적어도 우리가 만드는 디자인은 **인간적이고 영혼이 있어야 한다**는 것이 그녀의 철학입니다.

완전 동의하긴 어렵지만, 한 가지는 확실합니다. AI에게 디자인을 "시키는" 것과 AI와 "함께" 디자인하는 것은 다릅니다. 기준을 세우고, DNA를 주입하고, 반복 개선하는 과정 — 결국 **디자인 의사결정은 여전히 사람의 몫**입니다.

## 정리

| 원칙 | 핵심 |
|------|------|
| 디자인 시스템 | 색상, 폰트, 버튼, 간격의 일관된 기준을 먼저 세워라 |
| 평균 탈출 | 뻔한 폰트, 뻔한 색상을 의도적으로 피하라 |
| DNA 주입 | 한 번 만든 디자인 표준을 모든 프로젝트에 적용하라 |
| 콘텐츠 연결 | "무엇을 보여주는가"와 "어떻게 보여주는가"를 연결하라 |
| 섹션별 분해 | 통으로 요청하지 말고 섹션 단위로 접근하라 |
| 반복 개선 | 5~6번 다듬으면 품질이 극적으로 올라간다 |
| 도구 조합 | 단일 도구보다 여러 도구의 조합이 효과적이다 |
| 감각 축적 | Mobbin, Dribbble을 꾸준히 브라우징하라 |
