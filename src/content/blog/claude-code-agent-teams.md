## AI 코딩의 다음 단계

ChatGPT, Claude 같은 AI 챗봇으로 코드를 작성하는 건 이제 익숙합니다. 하지만 대부분의 AI 코딩은 "질문 → 답변" 방식의 1:1 대화입니다. 복잡한 프로젝트를 진행하다 보면 한계가 느껴집니다.

> "프론트엔드 작업하는 동안 백엔드도 같이 진행됐으면..."

이 생각을 현실로 만든 것이 **Claude Code의 에이전트 팀(Agent Teams)** 기능입니다.

## Claude Code란

Claude Code는 Anthropic에서 만든 CLI 기반 AI 코딩 도구입니다. 에디터 플러그인이 아니라 터미널에서 직접 실행됩니다.

```bash
# 설치
npm install -g @anthropic-ai/claude-code

# 실행
claude
```

일반적인 AI 코딩 도구와의 차이점:

- **파일 시스템 직접 접근**: 코드를 복사/붙여넣기할 필요 없이, AI가 직접 파일을 읽고 수정
- **터미널 명령 실행**: 빌드, 테스트, git 작업을 AI가 직접 수행
- **프로젝트 컨텍스트 이해**: 전체 프로젝트 구조를 파악한 상태에서 작업

이것만으로도 강력한데, 에이전트 팀은 여기서 한 단계 더 나아갑니다.

## 에이전트 팀이란

에이전트 팀은 **여러 Claude Code 인스턴스를 팀으로 조율**하는 기능입니다. 한 세션이 리더 역할을 하고, 나머지 팀원들이 각자 독립적으로 작업합니다.

기존의 서브에이전트(subagent)와 비교하면 차이가 명확합니다.

| | 서브에이전트 | 에이전트 팀 |
|---|---|---|
| **컨텍스트** | 메인 에이전트에게 결과 반환 | 완전히 독립적인 컨텍스트 |
| **소통** | 메인 에이전트에게만 보고 | 팀원끼리 직접 메시지 교환 |
| **조율** | 메인 에이전트가 전부 관리 | 공유 작업 목록으로 자체 조율 |
| **비유** | 심부름 시킨 후 결과 받기 | 팀 프로젝트에서 역할 분담 |

서브에이전트는 "이거 조사해와"라고 시키고 결과를 받는 느낌이라면, 에이전트 팀은 진짜 팀원들이 각자 맡은 일을 하면서 필요하면 서로 대화하는 느낌입니다.

## 활성화 방법

에이전트 팀은 아직 실험적 기능이라 수동으로 활성화해야 합니다.

```json
// ~/.claude/settings.json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

그리고 분할 창 모드를 사용하려면 tmux가 필요합니다. WSL 환경이라면 [이전 글](/blog/wsl-tmux-setup)에서 다룬 세팅이 그대로 적용됩니다.

## 실전 사용법

### 팀 시작하기

Claude Code에서 자연어로 팀을 만들 수 있습니다.

```
블로그 프로젝트를 리팩토링하자. 에이전트 팀을 만들어서:
- 팀원 1: 컴포넌트 구조 개선
- 팀원 2: 스타일 시스템 정리
- 팀원 3: SEO 최적화
각자 작업하고 결과를 공유해.
```

이렇게 요청하면 Claude가 팀을 생성하고, 각 팀원에게 작업을 할당합니다. 팀원들은 tmux 분할 창에서 각각 독립적으로 동작합니다.

### 팀원과 직접 대화

리더를 거치지 않고 개별 팀원과 직접 소통할 수 있습니다.

- **tmux 분할 창 모드**: 해당 팀원의 창을 클릭해서 직접 입력
- **in-process 모드**: `Shift+Down`으로 팀원 간 전환

예를 들어, SEO 담당 팀원에게 "메타 태그 말고 구조화된 데이터도 추가해줘"라고 직접 지시할 수 있습니다.

### 작업 목록으로 조율

팀은 공유 작업 목록을 통해 조율됩니다. 작업에는 세 가지 상태가 있습니다.

- **대기 중**: 아직 시작 안 함
- **진행 중**: 팀원이 작업 중
- **완료**: 작업 완료

작업 간 종속성도 설정할 수 있어서, "API 구현이 끝나야 프론트엔드 연동 시작" 같은 순서를 자동으로 관리합니다.

## 어떤 작업에 효과적인가

에이전트 팀은 모든 상황에 적합한 건 아닙니다. 조율 오버헤드가 있고 토큰 비용이 팀원 수만큼 늘어나기 때문에, 병렬 작업이 실질적인 가치를 만드는 경우에 사용해야 합니다.

### 잘 맞는 경우

**코드 리뷰를 여러 관점에서 동시에**

```
PR #142를 리뷰할 에이전트 팀을 만들어:
- 보안 관점 리뷰어
- 성능 관점 리뷰어
- 테스트 커버리지 리뷰어
각자 리뷰하고 발견한 것을 보고해.
```

한 명이 순차적으로 보는 것보다 빠르고, 관점이 분리되어 있어서 더 철저합니다.

**새 기능을 계층별로 분담**

프론트엔드, 백엔드, 테스트를 각각 다른 팀원이 담당합니다. 파일이 겹치지 않기 때문에 충돌 없이 병렬 진행이 가능합니다.

**디버깅할 때 여러 가설을 동시에 검증**

```
앱이 한 번 메시지 보내면 연결이 끊기는 버그가 있어.
5명의 팀원을 만들어서 각각 다른 가설을 조사해.
서로 대화하면서 상대방의 이론을 반박하도록 해.
```

순차적으로 하면 첫 번째 가설에 편향되기 쉬운데, 동시에 여러 방향을 탐색하면 진짜 원인을 더 빨리 찾을 수 있습니다.

### 안 맞는 경우

- **같은 파일을 여러 팀원이 수정**: 덮어쓰기 충돌 발생
- **순서가 중요한 작업**: A가 끝나야 B를 시작할 수 있는 경우
- **단순한 작업**: 조율 오버헤드가 이점보다 큼

## 실전 팁

### 적절한 팀 크기

대부분의 작업에서 **3~5명**이 적당합니다. 팀원이 많을수록 토큰 비용이 선형으로 증가하고, 조율 오버헤드도 커집니다. 세 명이 집중하는 게 다섯 명이 산만한 것보다 낫습니다.

### 팀원당 작업 5~6개

팀원 하나에 작업을 5~6개 정도 배분하면 유휴 시간 없이 생산적으로 돌아갑니다. 15개의 독립적인 작업이 있다면 3명의 팀원이 좋은 출발점입니다.

### 파일 충돌 방지

가장 중요한 규칙입니다. 두 팀원이 같은 파일을 건드리면 나중에 쓴 쪽이 이전 작업을 덮어씁니다. 작업을 분배할 때 **파일 소유권**을 명확히 나누세요.

### 리더가 직접 작업하지 않게 하기

가끔 리더 에이전트가 팀원을 기다리지 않고 직접 코딩을 시작할 때가 있습니다. 이때는 이렇게 말하면 됩니다.

```
팀원들이 작업을 끝낼 때까지 기다려. 직접 구현하지 마.
```

## Windows에서 사용하기

Windows에서 에이전트 팀을 쓰려면 다음 세팅이 필요합니다.

```
Windows Terminal → WSL2 (Ubuntu) → tmux → Claude Code
```

1. **Windows Terminal + WSL2 설치**: [이전 글](/blog/wsl-tmux-setup) 참고
2. **tmux 설치**: `sudo apt install tmux -y`
3. **Claude Code 설치**: `npm install -g @anthropic-ai/claude-code`
4. **에이전트 팀 활성화**: settings.json에 환경 변수 추가
5. **tmux 세션에서 Claude Code 실행**: `tmux new -s dev` → `claude`

tmux 세션 안에서 Claude Code를 실행하면 자동으로 분할 창 모드가 활성화됩니다. 각 팀원이 자기만의 창을 가지고 동시에 작업하는 모습을 볼 수 있습니다.

## 마무리

에이전트 팀은 AI 코딩의 패러다임을 바꾸는 기능입니다. "AI에게 물어보고 답을 받는" 수준을 넘어서, **AI가 팀으로 일하는** 시대가 시작되었습니다.

아직 실험적 기능이라 제한사항이 있지만, 병렬 작업이 필요한 프로젝트에서는 이미 실용적으로 쓸 수 있습니다. WSL + tmux 세팅이 되어 있다면, 지금 바로 시도해보세요.

```bash
# tmux 세션 시작
tmux new -s project

# Claude Code 실행
claude

# "에이전트 팀을 만들어서..." 라고 입력하면 끝
```
